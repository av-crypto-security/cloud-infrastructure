# Object Storage Trigger Pipeline

This module implements an event-driven serverless pipeline using Object Storage
as an event source.

## Overview

The system is composed of two serverless functions and a storage trigger:

- Upload Function (`cloud-function`)  
  Generates a file and uploads it to Object Storage.

- Object Storage Bucket  
  Acts as a persistence layer and event source.

- Trigger Function (`trigger-function`)  
  Processes events generated by object creation.

## Architecture

The pipeline follows an asynchronous event-driven model:

1. The upload function is invoked manually or via HTTP.
2. A file is generated and uploaded to Object Storage.
3. Object Storage emits a `create-object` event.
4. The trigger invokes the second function.
5. The event payload is processed and logged.

## Object Storage Integration

The upload function uses the S3-compatible API via `boto3`.

Authentication is performed using access keys associated with a service account.

The function lifecycle:

- create temporary file
- upload to bucket
- remove local file

## Configuration

The following environment variables are required:

- `ACCESS_KEY` — S3 access key  
- `SECRET_KEY` — S3 secret key  
- `BUCKET_NAME` — target bucket  
- `TIME_ZONE` — optional, defaults to UTC  

## Trigger Behavior

The trigger listens for create-object events in the configured bucket.

Each event contains metadata about the uploaded object and is passed
to the trigger function.

## Use Cases

This pattern can be used in:

- log processing pipelines  
- file ingestion systems  
- asynchronous data processing  
- event-driven microservices  

## Notes

- The system is fully decoupled via events.
- Functions are stateless.
- Object Storage acts as both storage and event bus.
